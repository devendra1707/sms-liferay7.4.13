Converting a Service Builder Module from Spring DI to OSGi DS

Upgrading Service Builder Services

Step 2 of 3

Prior to the new DXP version, Service Builder modules could only use Spring for dependency injection (DI). Now OSGi Declarative Services (DS) is the default DI mechanism for new Service Builder modules. Although OSGi DS is the default DI mechanism, Spring is still supported. Therefore, this is an optional migration step.

To learn more about the decision to convert your Service Builder modules’ DI mechanism and how to complete the conversion process, see the Migrating a Service Builder Module from Spring DI to OSGi DS article.
